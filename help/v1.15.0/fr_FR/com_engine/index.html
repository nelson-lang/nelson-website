Interface client du modèle d'objet composant (COM)

Le module COM Engine permet à Nelson d'interagir avec les composants Windows COM (Component Object Model) et ActiveX.

Il permet l'automatisation des applications logicielles, l'accès à leurs méthodes et propriétés, et la manipulation d'objets COM directement depuis les scripts Nelson.

Cela fournit un moyen d'intégrer Nelson avec une large gamme de logiciels Windows, y compris les applications Microsoft Office.

  • Exemples COM
    Quelques exemples pour montrer comment utiliser l'interface COM.
  • COM_delete
    Supprime un contrôle ou serveur COM.
  • COM_fieldnames
    Retourne les noms des propriétés d'un objet COM.
  • COM_get
    Obtenir la valeur d'une propriété depuis l'interface COM.
  • COM_invoke
    Invoquer une méthode sur un objet ou interface COM.
  • COM_ismethod
    Détermine si l'entrée est une méthode existante d'un objet COM.
  • COM_isprop
    Détermine si l'entrée est une propriété existante d'un objet COM.
  • COM_methods
    Retourne les noms des méthodes d'un objet COM.
  • COM_range
    Fonction privée : assistant de plage.
  • COM_set
    Définir la propriété d'un objet ou interface à la valeur spécifiée.
  • COM_used
    Retourne la liste des handles COM actuellement utilisés.
  • COM_xlsfinfo
    Détermine si le fichier contient une feuille de calcul Microsoft Excel.
  • COM_xlsread
    Lire un fichier de feuille de calcul Microsoft Excel en utilisant COM.
  • COM_xlswrite
    Écrire un fichier de feuille de calcul Microsoft Excel en utilisant COM.
  • actxGetRunningServer
    Handle vers une instance en cours d'exécution du serveur Automation.
  • actxcontrollist
    Obtenir les contrôles ActiveX disponibles.
  • actxserver
    Crée un serveur COM.
  • actxserverlist
    Obtenir les serveurs ActiveX disponibles.
  • iscom
    Détermine si l'entrée est un objet COM ou ActiveX.
